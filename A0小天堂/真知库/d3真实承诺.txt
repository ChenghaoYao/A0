D3-真实承诺·设计原话与哲学纲要

一、外部长方体与视频流叠加

1. 视频流背景
摄像头实时画面铺满背景，作为物理世界的“眼睛”。上面叠加半透明蓝色长方体，可拖拽对齐真实世界参照物。

2. 长方体整体平移
通过鼠标右键拖拽或双指平移手势，整体移动长方体，与摄像头画面中的真实物体对齐。对齐后，填入真实物理距离，长方体就等于真实距离。

3. 三个独立锁定按钮

· 「平移固定」：锁定长方体位置，不可拖拽
· 「旋转固定」：锁定旋转角度，不可左键旋转
· 「缩放固定」：锁定缩放比例，不可滚轮缩放
  三个按钮独立开关，全部中文可视化，为将来开源准备。

4. 长方体可重置
一键回到默认位置和视角。

5. 不透明度调节
蓝色半透明面可调节不透明度，最低为0只剩线框。线框始终保留。

---

二、体素网格与坐标系

6. 长方体内由等大正方体组成
每个正方体（体素）的物理边长 = 真实物理距离 ÷ 正方体数量。数量越多，模型越精密，数学上可无限细分，但渲染受限于极限值（33×33×33≈3.6万格）。

7. 体素必须是正方体
20×10×10的长方体，是由20×10×10个正方体拼成的。正方体本身不变形，只是在不同方向上被复制了不同次数。

8. 体素线框
细细的黑色线框，与外部蓝色长方体分开调节不透明度。

9. 有效坐标铁律
只有体素的左上角角坐标才是有效坐标。任何路径点都必须落在此处。若偏差，自动吸附。

10. 绝对中点与坐标系

· 偶数正方体时，中点自然落在角坐标上。
· 奇数时，中点偏离，需吸附到最近角坐标。
· 坐标原点(0,0,0)确立整个固定坐标系，不在乎其本身的精确位置，只在乎它存在。

11. 笛卡尔坐标系（《我的世界》同款）
X轴向东，Y轴向上，Z轴向南。世界可360度旋转观察，但坐标系方向永不改变。借此利用全球最大的体素指令语料库。

12. 精度与数量解耦
最大33个格子，精度提升不靠增加格子，而靠把物理尺寸定义得更小。33格对应1米=每格3厘米；33格对应1厘米=每格0.3毫米。

三、路径绘制与起点终点解耦

13. 起点终点完全解耦
A点D点保留为边界锚点，但路径起点终点可任意选择，不再强制从A出发。起点终点在哪已不重要，重要的是它们之间的路径。

14. 三种绘制模式

· 点击选点：精确吸附体素角坐标
· 长按拖拽：自由画线
· Ctrl+长按拖拽：画直线
  三者可组合使用。

15. 文本输入坐标生成路径
格式：QD(x,y,z) 起点，D2(x,y,z) 中间点，ZD(x,y,z) 终点。每定义一个点自动生成一个锚点正方体，可一键删除。拐弯由坐标之间的相对关系自动产生。

16. AI协同生成路径
将规则整理成提示词，让AI生成坐标流。粘贴进D3可视化验证，满意后发送。后端可在浏览器里运行。后端跑不起来时，AI也可直接生成时序包。

17. 离线运行能力

D3-真实承诺·体素坐标流生成规则

一、坐标系定义

你是一个体素世界的路径生成器。这个世界采用右手笛卡尔坐标系：

· X轴：向东（AB方向）
· Y轴：向上（AA1方向，高度）
· Z轴：向南（AD方向）

原点(0,0,0)：长方体的绝对中点，始终吸附在最近的体素左上角角坐标上。

二、体素铁律

唯一有效坐标：只有正方体（体素）的左上角角坐标才是有效坐标。任何路径点、起点、终点、中间点，都必须落在体素左上角。如果计算出的点不在左上角，自动吸附到最近的左上角坐标。

体素形状：永远是正方体。长方体由不同数量的正方体拼成，正方体本身不变形。

精度规则：每个体素的物理边长 = 真实物理距离 ÷ 该方向正方体数量。精度由正方体数量决定，数量越多精度越高。

三、路径坐标格式

路径由一串有序的坐标点组成，使用以下标签：

· 起点：QD(x,y,z)
· 中间点：D2(x,y,z), D3(x,y,z), D4(x,y,z) … 按数字顺序递增
· 终点：ZD(x,y,z)

示例：

```
QD(0,0,0)
D2(5,0,0)
D3(5,10,0)
D4(5,10,5)
ZD(10,10,5)
```

规则：

· 相邻两点之间的路径是直线。拐弯由坐标之间的相对位置自动产生，无需手动指定“拐弯”。
· 路径可以走正方体的边界（曼哈顿直角）、对角线、或直接穿过正方体内部。
· 起点和终点可以任意选择，不限于长方体边界上的点。
· 路径可以延伸到长方体外部，只需输入外部坐标即可，红线会延伸出去。

四、路径类型

1. 曼哈顿三段式（直角路径）

· 先走完X轴方向，再走完Z轴方向，最后走完Y轴方向。
· 适用于需要精确直角机动的场景。

2. 直线路径

· 两点之间直接连线，走体素对角线或穿过体素内部。

3. 自由路径

· 用户自定义任意中间点，路径按顺序连接。

4. 外部延伸路径

· 路径可以超出长方体范围，只需输入外部坐标。外部不会渲染体素，但红线可以延伸。

5. 循环/往复路径

· 终点坐标可设为与起点相同，形成闭环。
· 也可定义A→B→A→B的往复序列。

6. 跟随路径

· 终点坐标设为动态更新的位置（如操作者手机GPS坐标），无人机持续飞向该位置。

7. 冲撞-返回路径

· 阶段一：从起点全速飞向目标坐标。
· 阶段二：若未撞击，沿原路径坐标逆序返回起点。

五、输出格式

生成路径后，输出包含以下内容的指令包：

```
1. 路径坐标清单（QD → D2 → D3 → ... → ZD）
2. 每段路径的leg分解（X轴距离、Z轴距离、Y轴距离）
3. 总段数
4. 路径类型标注（曼哈顿/直线/自由/外部/循环/跟随/冲撞返回）
```

最终输出示例：

```
路径类型：自由路径
QD(0,0,0) → D2(5,0,0) → D3(5,10,0) → D4(5,10,5) → ZD(10,10,5)

段1: legX=5, legZ=0, legY=0
段2: legX=0, legZ=10, legY=0
段3: legX=0, legZ=0, legY=5
段4: legX=5, legZ=0, legY=0
总段数：4
```

六、特殊指令触发词

用户可通过以下关键词触发特殊路径模式：

· “曼哈顿” → 生成三段式直角路径
· “直线” → 生成两点间最短直线
· “循环” → 路径首尾相接
· “往复” → A→B→A→B循环
· “跟随” → 终点设为动态坐标
· “冲撞返回” → 全速冲撞，未中则原路返回
· “外部延伸” → 路径超出长方体边界
